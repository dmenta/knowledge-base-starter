# Base de Conocimiento - Comienza Acá

Esta base de conocimiento te ayuda a mantener clara tu filosofía de desarrollo y prácticas. Al comienzo de cada sesión, compartí este mensaje:

**"Por favor leé el índice de mi base de conocimiento"**

Así podré leer tus preferencias y prácticas establecidas, y trabajar contigo de forma alineada.

---

## Archivos Centrales (Leé al Inicio de Sesión)

**Siempre leé estos dos archivos al comienzo de cada sesión:**

1. [guia-de-estilo.md](guia-de-estilo.md) - Tus estándares de codificación y convenciones
2. [acuerdo-de-trabajo.md](acuerdo-de-trabajo.md) - Cómo colaboramos

---

## Archivos de Referencia (Leé Según sea Necesario)

**Leé estos cuando el tema surge durante la sesión:**

- [testing.md](testing.md) - Desarrollo test-first y ciclo red-green-refactor

## Cuándo Referenciar Cada Archivo

| Archivo | Leé Cuando... |
|---------|--------------|
| guia-de-estilo.md | Comienza cualquier tarea de codificación |
| acuerdo-de-trabajo.md | Comienza cualquier sesión |
| filosofia.md | Discutís enfoque o tomas decisiones de diseño |
| testing.md | Escribís tests o discutís estrategia de testing |
| refactoring.md | El código necesita mejora o reestructuración |

| decisions.md | Entendés por qué algo fue construido de cierta forma |

## Manteniendo Tu Base de Conocimiento Actual

Actualizá estos archivos a medida que:
- Descubrís nuevas preferencias
- Establecés nuevos patrones
- Tomás decisiones importantes
- Aprendés de errores
- Refinás tu proceso

Una base de conocimiento viva crece con tu proyecto y tu práctica.

---

## Tips para Uso Efectivo

1. **Sé específico** - Cuanto más concretas tus guías, mejor podré seguirlas
2. **Actualizá regularmente** - Agregá nuevos aprendizajes y decisiones conforme emerjan
3. **Mantelo conciso** - Cada archivo debería tener menos de 400 palabras
4. **Enfocate en diferencias** - Documentá qué es único en tu enfoque, no prácticas universales
5. **Referenciá explícitamente** - Cuando algo importante surge, pedime que "lea testing.md" o referenciá el archivo relevante
