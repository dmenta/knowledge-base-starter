# Guía de Uso y Mantenimiento

Esta base de conocimiento es un documento vivo que crece con nuestro proyecto y entendimiento. Acá está cómo mantenerla actual y valiosa.

---

## Cuándo Actualizar

Actualizá la base de conocimiento cuando:
- **Descubrís nuevas preferencias** - "Ahora siempre hacemos X de esta forma"
- **Estableces nuevos patrones** - Un patrón emergió y querés repetirlo
- **Tomas decisiones importantes** - Registralas en [decisions.md](decisions.md)
- **Aprendés de errores** - Actualizá guías para prevenir problemas repetidos
- **Cambian prácticas** - La guía antigua ya no aplica

---

## Cómo Actualizar

1. **Hacé el cambio** - Editá el/los archivo/s relevante/s
2. **Mantelo conciso** - Cada archivo debería mantenerse bajo 400 palabras
3. **Sé específico** - Ejemplos concretos sobre principios abstractos
4. **Commiteá con contexto** - Explicá el por qué en el mensaje de commit
5. **Compartí con el equipo** - Anunciá cambios significativos

---

## Guías de Archivos

**Cada archivo debería:**
- Enfocarse en un tema
- Ser rápidamente hojeablе (usá headers, listas, tablas)
- Referenciar archivos relacionados
- Incluir ejemplos cuando sea útil
- Mantenerse bajo 400 palabras (excepciones permitidas para decisions.md)

**Evitá:**
- Duplicación entre archivos
- Detalles de implementación que cambian frecuentemente
- Sintaxis específica de lenguaje (a menos que sea un proyecto específico de lenguaje)
- Consejo demasiado general que aplique a todo software

---

## Revisando Cambios

Cuando alguien actualiza la base de conocimiento:
- Leé los cambios cuidadosamente
- Discutí si no estás de acuerdo o tenés preguntas
- Proponé alternativas si ves una forma mejor
- Aprobá una vez que se alcance consenso

Esto no es solo documentación—es nuestro entendimiento compartido. Tratalo seriamente.

---

## Mantenimiento

**Mensualmente:**
- Revisá todos los archivos para precisión
- Remové guía desactualizada
- Agregá nuevos aprendizajes del trabajo reciente

**Cuando se onboardea:**
- Nuevos miembros del equipo deberían leer toda la base de conocimiento
- Sus preguntas revelan vacíos o secciones poco claras
- Actualizá basado en su feedback

**Cuando cambia la dirección:**
- Marcá decisiones supersedidas en decisions.md
- Actualizá archivos afectados
- Considerá agregar un ADR explicando el cambio

---

## ¿Preguntas?

Si no estás seguro si algo pertenece a la base de conocimiento, preguntate:
- **¿Es única la forma en que lo hacemos?** → Probablemente sí
- **¿Lo referenciamos repetidamente?** → Probablemente sí
- **¿Es obvio o universal?** → Probablemente no
- **¿Cambia frecuentemente?** → Probablemente no (o ponelo en otro lado)

Cuando tengas dudas, proponlo y discutilo con el equipo.

---

*Esta base de conocimiento representa nuestra sabiduría colectiva. Mantela actual, mantela concisa, y mantela honesta.*
